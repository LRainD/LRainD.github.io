## 1. 产品概览

### 1.1 业务背景与痛点

**业务现状**
数字合规官核心风控检测能力已完成落地，可实现招投标全流程 13 个核心节点的合规风险自动化校验，已建立完善的组织级开关、权限管控、风控规则引擎与检测结果输出体系。

**核心痛点**

| 角色/场景 | 痛点描述                                                         |
| ----- | ------------------------------------------------------------ |
| 用户侧   | 采购经办人查看风控结果入口深，无法一键获取全节点风险概览，对风险定位、报告查看的便捷性需求强烈              |
| 研发侧   | 团队暂不支持复杂的生成式大模型对话开发，需优先落地轻量、可控、可快速迭代的 MVP 版本，同时为后续能力升级预留架构空间 |
| 业务侧   | 传统功能式入口无法满足用户个性化查询需求，需通过对话式交互降低合规功能的使用门槛，提升风控整改效率            |

**AI 能力引入的必要性**
通过自然语言交互实现「指令 - 动作」的快速闭环，替代传统多层级页面点击操作，同时为后续风控结果解读、合规规则咨询、整改建议生成等深度智能化能力提供基础载体。

### 1.2 核心目标

**一期 MVP 核心目标**

| 目标类型 | 具体内容                                             |
| ---- | ------------------------------------------------ |
| 业务目标 | 落地轻量化对话式交互入口，实现「合规风险一键查看」核心闭环，复用现有业务流程模块接口       |
| 体验目标 | 用户获取全节点合规风险的操作路径从≥3 步缩短至 1 步，风险查看效率提升 70% 以上     |
| 架构目标 | 建立标准化的对话系统框架，为后续升级生成式 AI 问答、多轮对话、深度合规解读能力预留完整扩展位 |

**二期及长期目标**

- 从「指令导航助手」升级为「全场景合规智能助手」，实现风控结果解读、合规规则咨询、文本合规自检、整改建议生成等核心能力
- 从关键词匹配升级为自然语言意图识别，支持用户泛化表述的精准识别，意图识别准确率≥90%
- 实现对话历史服务端存储、多轮上下文对话、个性化合规建议等进阶能力

### 1.3 范围定义

| 一期 MVP 范围（必须落地）                                                                                      | 非一期范围（二期及以后落地，预留扩展位）                                                         | 永久禁止范围（红线规则）                                                                         |
| ---------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| 1. 侧边栏 AI 入口与弹窗聊天框基础交互2. 单意图识别（查看当前项目合规风险），支持关键词匹配触发3. 全节点合规风险卡片结构化渲染4. 固定节点「查看报告」快捷跳转5. 全量非合规意图固定兜底 | 1. 生成式 AI 问答、风控结果解读、整改建议生成2. 自然语言泛化意图识别3. 多轮对话与上下文管理4. 服务端对话历史存储5. 合规规则知识库问答 | 1. 任何与招投标 / 合同合规无关的问答响应2. 违反招投标法律法规的内容输出3. 跨组织、跨项目、跨节点的非授权数据查询4. 任何形式的通用闲聊、非业务相关内容响应 |

## 2. 目标用户与核心场景

### 2.1 目标用户与权限规则

| 用户角色    | 权限准入规则（必须同时满足）                                                | 核心需求                             |
| ------- | ------------------------------------------------------------- | -------------------------------- |
| 采购项目经办人 | 1. 所在采购组织已在运营后台开启数字合规官功能，且状态为启用；2. 为当前采购项目的直接经办人              | 一键查看当前项目全节点合规风险，快速定位预警项，查看详细检测报告 |
| 关联合规人员  | 1. 所在采购组织已在运营后台开启数字合规官功能，且状态为启用；2. 为运营后台在数字合规官配置中，为该组织关联的合规人员 | 快速查看所负责项目的全节点合规风险概览，跟进风险处置进度     |

### 2.2 核心用户场景

**场景 1：一键查看全节点合规风险（主场景）**

- **前置条件**：用户已进入已开启数字合规官的采购项目业务节点页面，具备访问权限
- **用户旅程**：用户点击侧边栏数字合规官头像打开聊天框 → 点击【查看合规风险】快捷按钮 / 输入风险相关关键词 → AI 识别有效意图，拉取当前项目全节点风控数据 → 对话区输出结构化合规风险卡片 → 用户展开节点查看子项详情，点击【查看报告】获取该节点完整检测报告
- **后置条件**：用户完成风险查看，可基于报告进行内容整改

**场景 2：非合规意图兜底处理**

- **前置条件**：用户打开聊天框，输入非合规风险相关内容 / 点击非有效意图快捷按钮
- **用户旅程**：用户点击【项目状态查询】按钮 / 输入非白名单关键词的内容 → AI 触发兜底规则 → 输出固定兜底话术，明确告知能力范围，不做任何超出边界的响应
- **后置条件**：用户明确功能边界，切换至有效意图操作

## 3. 核心 AI 能力定义

### 3.1 一期 MVP AI 能力边界

#### 3.1.1 核心能力（仅支持 1 项主能力）

单意图精准响应能力：仅支持「查看当前项目全节点合规风险」这唯一有效意图，通过固定快捷按钮 + 预设关键词匹配两种方式触发，触发后自动拉取当前项目风控数据，输出标准化结构化风险卡片，不做任何生成式内容创作、开放式问答、多轮对话。

#### 3.1.2 能力触发规则

仅以下两种方式可触发有效能力，其余所有操作全部触发兜底策略：

- **快捷按钮触发**：用户点击聊天框底部固定的【查看合规风险】快捷按钮，直接触发有效动作
- **关键词匹配触发**：用户输入的文本内容中，命中预设关键词白名单中的任意一个关键词，即可触发有效动作

一期预设关键词白名单：风险、合规、问题、哪里错了、不通过、预警、检测结果（支持运营后台后续新增 / 修改，无需迭代代码）。

#### 3.1.3 绝对禁止能力（红线规则，研发必须 100% 严格执行）

- 禁止响应任何与「查看当前项目合规风险」无关的用户意图，包括但不限于项目状态查询、流程操作咨询、通用闲聊等
- 禁止任何生成式文本创作，所有响应内容必须为预设固定话术 / 现有接口拉取的标准化风控数据，禁止大模型自由生成内容
- 禁止调用任何当前项目以外的业务数据、风控数据，禁止跨组织、跨项目、跨节点的数据查询
- 禁止任何违反招投标法律法规、合规管理要求的内容输出

### 3.2 二期及未来 AI 能力规划（预留扩展位）

- **意图识别能力升级**：从关键词匹配升级为基于 NLU 的自然语言意图识别，支持用户泛化表述的精准识别，覆盖「规则解读、风险解析、整改建议、文本自检」等多类核心意图
- **生成式问答能力**：基于招投标合规知识库 + 当前项目风控数据，实现合规问题的精准生成式解答，包括风险原因解读、整改建议生成、合规规则释义等
- **多轮对话能力**：支持上下文关联的多轮对话，可针对单个风险点进行连续追问、深度答疑，实现槽位填充与意图流转管理
- **文本自检能力**：支持用户主动发起表单文本合规自检，提前识别潜在合规风险，给出优化建议

## 4. 对话系统核心规则

### 4.1 对话生命周期管理

- **会话初始化**：用户每次打开聊天框，视为一次全新会话，自动清空上一次对话历史，仅展示动态欢迎语
- **会话存续期**：仅在当前页面不刷新、不关闭的情况下，保留本次会话的对话历史（前端内存级存储）
- **会话终止**：页面刷新、关闭标签页、切换项目、关闭聊天框超过 5 分钟重新打开，会话自动终止，历史对话全部清空

### 4.2 输入输出规范

#### 4.2.1 用户输入规则

- **输入格式**：支持多行文本输入，最大输入长度 500 字，超出后无法继续输入
- **输入过滤**：禁止输入敏感词、违规内容，命中敏感词时提示「输入内容包含违规信息，请重新输入」
- **空输入处理**：输入内容为空时，发送按钮置灰不可点击，按下回车键无响应

#### 4.2.2 AI 输出规则

| 输出类型    | 输出规范                                                                                                                                                                                       | 触发场景                             |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------- |
| 动态欢迎语   | 1. 无风险项目：您好，我是您的数字合规官。今天是 {{当前日期}}，请问有什么可以帮您？您可以点击【查看合规风险】获取项目合规情况。2. 存在风险项目：您好，我是您的数字合规官。检测到您当前项目存在合规风险，您可以点击【查看合规风险】查看详情。3. 检测中项目：您好，我是您的数字合规官。当前项目部分节点正在进行合规检测，您可以点击【查看合规风险】查看已完成的检测结果。 | 每次打开聊天框，会话初始化时输出                 |
| 结构化风险卡片 | 严格按照通用渲染规则输出，适配所有采购类型的业务节点，包含节点名称、节点状态、风险等级、子项详情、查看报告按钮，具体规则见 5.2 节                                                                                                                        | 有效意图触发后，成功拉取到风控数据时输出             |
| 固定兜底话术  | 我尚未掌握此信息。作为数字合规官，我主要协助识别与应对数据、隐私及监管相关的合规风险                                                                                                                                                 | 所有非有效意图触发时，100% 输出该固定话术，无任何个性化修改 |
| 异常提示话术  | 1. 网络异常：「网络异常，无法获取合规数据，请稍后重试」2. 无数据：「当前项目暂无合规检测数据」3. 无权限：「您暂无当前项目的合规数据查看权限」                                                                                                                | 对应异常场景触发时输出                      |

### 4.3 兜底策略与异常处理

#### 4.3.1 全量兜底策略（无任何例外）

- 所有不在有效触发规则内的用户操作，包括但不限于输入非白名单关键词、点击非有效意图按钮、询问超出能力边界的问题，100% 触发固定兜底话术，不做任何其他处理
- 兜底话术仅支持固定文本输出，禁止任何泛化、生成式修改

#### 4.3.2 系统异常处理

| 异常场景            | 处理策略                             |
| --------------- | -------------------------------- |
| 风控数据接口请求失败 / 超时 | 解除加载态，展示异常提示话术，不崩溃、不白屏，用户可重新点击触发 |
| 接口返回空数据         | 展示「当前项目暂无合规检测数据」固定话术，不展示空卡片      |
| 用户权限失效          | 自动关闭聊天框，隐藏侧边栏入口，提示「您暂无该功能的访问权限」  |
| 网络中断后恢复         | 保留当前会话页面，用户可重新点击按钮发起请求，无需刷新页面    |

## 5. 交互与功能需求

### 5.1 侧边栏 AI 入口

#### 5.1.1 显隐规则

同时满足以下 2 个条件时展示，否则完全隐藏：

- 登录用户所在采购组织已开启数字合规官功能，且状态为启用
- 登录用户为当前项目经办人，或为该组织配置的关联合规人员

#### 5.1.2 展示与交互规则

- **展示位置**：业务页面右侧固定悬浮，不遮挡表单核心操作区，层级高于页面普通元素，低于弹窗
- **展示内容**：复用运营后台配置的数字合规官头像，无配置时展示系统默认头像
- **状态提示**：当前项目存在未查看的预警 / 拦截风险时，头像右上角展示红色未读小圆点，用户打开聊天框查看后红点消失
- **交互规则**：点击头像弹出 / 关闭聊天框，鼠标悬停展示 tooltip：「数字合规官 - 查看项目合规风险」，支持收起 / 展开缩略展示

### 5.2 聊天框主体与风险卡片渲染

#### 5.2.1 聊天框基础结构

采用「左对话区 + 右项目信息栏」固定布局，整体尺寸固定，支持拖动，不支持缩放：

- **顶部栏**：左侧展示数字合规官 logo + 在线状态，右侧关闭按钮，点击关闭聊天框
- **左侧对话区**：从上到下依次为对话历史展示区、底部输入框、固定快捷操作按钮区
- **右侧项目信息栏**：固定展示采购名称、采购编号、组织机构、采购方式、采购项目、采购品类、经办人，与当前项目强绑定，项目切换时自动更新

#### 5.2.2 快捷操作按钮规则

- 固定展示在输入框下方，全程可见，不随对话滚动
- 一期固定展示 2 个按钮：【查看合规风险】、【项目状态查询】
- 【查看合规风险】：点击后直接触发有效意图，输出风险卡片
- 【项目状态查询】：点击后触发兜底策略，输出固定兜底话术

#### 5.2.3 合规风险卡片通用渲染规则（适配全采购类型）

- **数据来源**：100% 复用现有业务流程节点接口、全节点风控检测结果接口，无新增接口开发
- **通用数据结构**：严格对齐接口返回的 node\_id、node\_name、node\_sort、node\_status、child\_node\_list、risk\_level、risk\_count、is\_support\_report 字段渲染，不做前端硬编码
- **排序规则**：严格按照接口返回的 node\_sort 升序排列，与业务系统采购流程进度顺序 100% 一致
- **展开规则**：默认所有节点折叠，存在预警 / 拦截风险的节点默认自动展开，用户可手动切换展开 / 折叠状态
- **【查看报告】按钮规则**：仅当 node\_name 为「报名与资审汇总、查看报价情况、报价汇总、成交报告」且 is\_support\_report=true 时展示，点击后弹出独立居中弹窗，调用现有接口展示对应节点的完整合规检测报告

## 6. 数据与模型要求

### 6.1 数据来源规范

- **权限数据**：来自运营后台数字合规官配置模块的组织开关、关联合规人员数据
- **项目基础数据**：来自招投标业务系统的项目详情接口
- **业务节点数据**：来自业务系统的项目流程节点接口
- **风控检测数据**：来自风控预警中心的全节点风控检测结果接口、合规报告详情接口
- 所有数据仅在当前项目会话内调用，禁止跨项目、跨组织数据调用

### 6.2 模型与调用要求（一期 MVP）

- **模型选型**：一期无需调用生成式大模型，仅需基于前端关键词匹配规则实现意图识别，最大化降低研发成本与调用风险
- **调用时序**：用户触发有效意图后，前端仅调用现有风控数据接口，拉取标准化数据后按预设规则渲染，无额外模型调用
- **性能要求**：接口请求响应时间≤2s，从触发意图到卡片渲染完成总耗时≤3s

### 6.3 二期模型预留要求

- 预留 RAG 知识库调用接口，支持后续接入招投标合规知识库
- 预留大模型生成式对话接口，支持 Prompt 工程配置、模型参数调整、输出内容审核
- 预留意图识别模型接口，支持多意图分类、实体抽取、槽位填充能力扩展

## 7. 不确定性管理与风险控制

### 7.1 核心不确定性与应对方案

| 不确定性场景                     | 风险等级 | 应对方案                                                                                   |
| -------------------------- | ---- | -------------------------------------------------------------------------------------- |
| 用户输入泛化表述未命中关键词白名单，无法触发有效动作 | 低    | 1. 一期优先保障快捷按钮触发，降低对关键词匹配的依赖；2. 兜底话术引导用户使用快捷按钮；3. 支持运营后台快速更新关键词白名单，无需迭代代码               |
| 接口返回数据结构不一致，导致卡片渲染异常       | 中    | 1. 提前统一节点数据通用结构，强制所有采购类型接口返回字段对齐通用结构；2. 异常字段做兼容处理，不影响整体卡片渲染                            |
| 后续升级生成式模型后，出现幻觉、违规内容输出     | 高    | 1. 一期建立严格的输出内容审核机制，预留内容安全过滤接口；2. 建立 Prompt 工程规范，严格限定模型输出范围；3. 建立兜底降级机制，模型异常时自动回退至规则化响应 |

### 7.2 内容安全与合规管控

- 建立输入输出双重敏感词过滤机制，禁止违规内容输入与输出
- 所有 AI 输出内容必须符合招投标法律法规、企业合规管理要求，禁止任何违规内容输出
- 建立全对话日志留存机制，所有用户输入与 AI 输出均需留存日志，支持后续审计与问题排查
- 严格遵守数据安全规范，所有项目数据、风控数据仅在当前会话内使用，禁止数据外泄、违规存储
