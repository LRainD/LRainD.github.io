# 数字合规官设置

## 📋 业务与功能

### 1.1 核心目标

为集采工作台提供数字合规官管理功能，允许被授权的管理员设置对应组织的合规人员，用于风控预警等业务的合规审核。

### 1.2 功能清单

- **[组织合规人员配置]**：为组织设置合规人员，支持多选
- **[配置列表管理]**：查看、新增、编辑、删除合规人员配置
- **[组织筛选]**：按组织机构筛选配置，支持包含下级
- **[权限控制]**：仅运营端配置的管理员可访问此页面

### 1.3 交互要点

- **权限校验**：进入页面时检查当前用户是否为管理员，非管理员显示无权限提示
- **新增配置**：选择组织 → 选择合规人员 → 保存
- **编辑配置**：修改合规人员 → 保存（组织不可修改）
- **删除配置**：二次确认后删除

### 2.2 数据来源

- **数据类型**：配置列表
- **数据源**：生成示例数据
- **关键字段**：
  - `id`: 配置ID
  - `orgName`: 组织名称
  - `orgValue`: 组织值
  - `compliancePersonnel`: 合规人员数组
  - `createTime`: 创建时间
  - `updateTime`: 更新时间
  - `creator`: 创建人
  - `updater`: 更新人

### 2.3 权限控制

- **管理员判断**：`currentUser.isAdmin` 控制页面访问权限
- **管理的组织**：`currentUser.managedOrgs` 可扩展用于细粒度权限控制

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `onConfigAdd` | `object` | 新增配置保存成功时 | 返回新增的配置记录 |
| `onConfigEdit` | `object` | 编辑配置保存成功时 | 返回编辑的配置记录 |
| `onConfigDelete` | `string` | 删除配置成功时 | 返回删除的配置ID |

### 5.2 动作列表（actionList）

| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `refreshList` | - | - | 刷新配置列表 |
| `openAddModal` | - | - | 打开新增配置弹窗 |
| `openEditModal` | `{ id: string }` | 配置ID | 打开编辑配置弹窗 |

### 5.3 变量列表（varList）

| 变量名称 | 类型 | 默认值 | 说明 |
|---------|-----|-------|------|
| `dataSource` | `array` | `[]` | 配置列表数据 |
| `isModalOpen` | `boolean` | `false` | 弹窗显示状态 |
| `modalType` | `string` | `'add'` | 弹窗类型（add/edit） |
| `editingRecord` | `object` | `null` | 当前编辑的记录 |

### 5.4 数据结构

```typescript
{
  id: string;                    // 配置ID
  orgName: string;               // 组织名称
  orgValue: string;              // 组织值
  compliancePersonnel: string[]; // 合规人员数组
  createTime: string;            // 创建时间
  updateTime: string;            // 更新时间
  creator: string;               // 创建人
  updater: string;               // 更新人
}
```
