# 企业关联检测配置页面 - 规格文档

## 概述

本文档记录企业关联检测配置页面的设计规格，重点描述企业名称模糊搜索功能的交互设计。

---

## 功能模块

### 1. 企业名称模糊搜索

#### 1.1 功能描述
用户在"企业名称"输入框中输入关键字后，系统提供智能模糊搜索建议，帮助用户快速选择企业并自动填充统一社会信用代码。

#### 1.2 交互流程

```
用户输入 → 防抖处理(500ms) → 触发搜索 → 展示下拉建议 → 用户选择 → 自动填充
```

#### 1.3 触发条件

| 条件 | 是否触发搜索 | 说明 |
|------|-------------|------|
| 输入为空 | ❌ 否 | 不显示下拉框 |
| 输入1个字符 | ❌ 否 | 字符数不足，不触发 |
| 输入2个及以上字符 | ✅ 是 | 满足最小长度要求 |
| 输入包含"公司" | ❌ 否 | 关键词过滤，避免无效搜索 |
| 输入包含"有限公司" | ❌ 否 | 关键词过滤，避免无效搜索 |

#### 1.4 搜索结果展示

**下拉框规格：**
- 位置：相对于输入框底部定位（fixed定位，突破表格overflow限制）
- 宽度：与输入框等宽
- 最大高度：320px
- 最小宽度：400px（支持水平滚动）
- z-index：9999（确保最上层显示）

**建议项规格：**
- 数量：固定返回10条模拟数据
- 数据结构：企业名称 + 统一社会信用代码（18位数字）

**企业名称生成规则：**
```
{城市}{关键字}{企业类型}{随机数}

示例：
- 北京腾讯科技有限公司1234
- 上海阿里巴巴网络科技5678
```

**城市池：** 北京、上海、深圳、广州、杭州、苏州、成都、武汉、南京、西安

**企业类型池：** 科技有限公司、股份有限公司、集团有限公司、实业有限公司、贸易有限公司、网络科技、信息技术、智能科技、建筑工程、物流运输

#### 1.5 视觉效果

**关键字高亮：**
- 匹配的关键字显示为红色 `#FF4D4F`
- 使用正则表达式匹配，不区分大小写

**悬浮效果：**
- 默认背景：白色
- 悬浮背景：浅蓝色 `#E6F4FF`
- 过渡：无过渡，即时响应

**选中效果：**
- 点击后下拉框关闭
- 企业名称填入输入框
- 统一社会信用代码自动填入对应字段

#### 1.6 防抖处理

- 延迟时间：500ms
- 实现方式：使用setTimeout + clearTimeout
- 触发时机：用户停止输入后500ms触发搜索

#### 1.7 边界处理

| 场景 | 处理方式 |
|------|---------|
| 页面滚动 | 下拉框跟随输入框位置更新 |
| 窗口resize | 下拉框位置重新计算 |
| 点击外部区域 | 下拉框关闭 |
| 输入框失焦 | 保持下拉框显示（除非点击外部）|
| 无搜索结果 | 显示"暂无数据" |
| 搜索中 | 显示"搜索中..."加载状态 |

## 组件设计

### EnterpriseSearchInput 组件

**Props：**
```typescript
interface EnterpriseSearchInputProps {
  value: string;                          // 当前输入值
  onChange: (value: string) => void;      // 输入变化回调
  onSelect: (name: string, creditCode: string) => void;  // 选择回调
}
```

**内部状态：**
```typescript
interface SearchState {
  inputValue: string;           // 输入框值
  suggestions: SearchSuggestion[];  // 搜索建议列表
  showDropdown: boolean;        // 是否显示下拉框
  loading: boolean;             // 是否加载中
  hoveredIndex: number;         // 当前悬浮的索引
}
```

**Refs：**
```typescript
inputRef: HTMLInputElement      // 输入框引用，用于定位
dropdownRef: HTMLDivElement     // 下拉框引用
debounceTimerRef: Timeout       // 防抖定时器
```

## 性能优化

1. **防抖处理**：500ms延迟，减少频繁搜索
2. **fixed定位**：避免表格overflow:hidden裁剪
3. **事件委托**：滚动和resize事件监听
4. **条件渲染**：不满足条件时不生成模拟数据

## 研发注意点

1. 实际开发时需替换 `generateMockEnterprises` 为真实API调用
2. 统一社会信用代码由后端返回，前端只做展示
3. 考虑增加搜索历史、热门企业等增强功能
4. 移动端适配需要调整下拉框宽度
5. 考虑键盘导航支持（上下箭头选择、回车确认）
