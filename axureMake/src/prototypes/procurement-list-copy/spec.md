# 编制采购清单页面规格文档

## 业务与功能
- **页面定位**：集采工作台的非招标采购流程节点，用于编制采购清单。
- **核心目标**：提供采购清单的录入、编辑、校验功能，并集成“数字合规官”进行智能风险检测。
- **功能清单**：
  - 侧边栏导航：展示系统菜单，支持收起/展开。
  - 顶部导航：展示当前用户、语言切换、快捷操作入口及通知。
  - 采购基本信息区：展示采购编号、组织机构、采购方式等。
  - 流程步骤条：展示采购发起到采购成交的5个环节，当前高亮“采购发起”。
  - 子流程节点进度图：展示该阶段内的多个子节点状态（如编制采购清单、发布公告等）。
  - 清单表格：支持多行数据展示、编辑、全选、添加附件等。
  - 建议与说明栏：私有库建议栏、供应商报价说明输入框。
  - 招募供应商开关：单选配置是否招募。
  - 智能合规检测：
    - 右侧悬浮面板：展示风险提示、数字合规官入口。
    - 聊天式交互弹窗：通过问答触发风险检查展示。
    - 拦截与检测中弹窗：在进行敏感操作（如“下一步”）时，触发合规检测流程展示。

## 内容规划
- **信息架构**：
  - **左侧**：深蓝色主题色侧边栏。
  - **顶部**：白色背景标题栏。
  - **主体内容区**：淡灰色背景，卡片式布局。
    - 面包屑导航与操作按钮区（上一步、保存、下一步）。
    - 倒计时卡片与基本信息网格。
    - 流程进度条（箭头样式）。
    - 节点步骤连线图。
    - 采购清单主表格区块（带工具栏：选择清单模板、查询历史价格、添加类别等）。
    - 底部配置项卡片（建议栏、报价说明、招募选项）。
- **数据来源**：
  - 静态 Mock 数据（如侧边栏菜单、基本信息、表格初始数据）。
  - 弹窗中内置静态 HTML 结构转换而来的 JSX 状态。

## 布局与结构
- **整体布局**：
  - `flex h-screen overflow-hidden` 主框架。
  - 左侧 `w-64` 固定宽度侧边栏（深色风格）。
  - 右侧纵向 `flex flex-col` 主体区，包含顶部 Header 和支持滚动的 Main 区域。
- **关键尺寸与响应式**：
  - 顶部 Header 高度 `h-14`。
  - 内容区卡片均有圆角 `rounded`、边框 `border` 和轻微阴影 `shadow-sm`，内部 `p-4` 或 `p-8` 边距。
  - 表格区域支持横向滚动 `overflow-x-auto`。
  - 弹窗采用居中布局，带半透明黑色遮罩 `bg-black/50 backdrop-blur-sm`，宽度限制如 `max-w-5xl` 或 `max-w-md` 等。

## 视觉规范
- **设计风格**：
  - 参考提供的 HTML 文件，主要使用 Tailwind CSS 构建。
  - 品牌色：主色 `primary` (#007ADD)，并辅以各种中性灰 `slate`/`gray`。
  - 图标：大量使用 Material Symbols Outlined 图标。
  - 暗色模式适配：代码结构保留原 HTML 的 `dark:bg-xxx` 等暗黑模式类名支持。
- **组件状态定义**：
  - 按钮：默认 `bg-primary`，悬停 `hover:bg-primary-dark`，禁用态样式未特殊指定但可扩展。
  - 输入框/多选框：获得焦点时 `focus:ring-primary` 或 `focus:border-primary`。
  - 流程条：激活态带有主色边框与高亮背景，箭头连缀。
  - 聊天弹窗：带淡入缩放动画 `animate-in fade-in zoom-in duration-300`。
