# AI智能报名文件检测 - 研发评审规格文档

## 📋 业务与功能

### 1.1 核心目标
在资格审查弹窗中增加「AI智能报名文件检测」模块，用于展示系统对供应商报名文件的自动检测结果。该功能在开启智能审查后，报名截止时自动触发计算；若采购发生变更，则在新的报名截止后重新计算。

### 1.2 功能清单

- **[AI智能报名文件检测展示]**：在资格审查弹窗中独立展示检测模块，包含检测项表格
- **[检测项可人工调整]**：重要程度和检测结果支持人工修改，并标注「（支持人工调整）」
- **[检测结果联动清空]**：检测结果从「待核实」改为「通过」或「未通过」时，异常建议列自动清空
- **[检测项移除]**：支持逐行移除检测项
- **[自动触发计算]**：报名截止后自动执行；采购变更后在新报名截止后重新计算
- **[附件智能检测]**：在「资审附件组成」表格中增加「附件智能检测」列，展示每个附件的AI检测结果（通过/异常）
- **[附件检测详情查看]**：检测状态为「异常」的附件，可点击「异常，查看详情」打开详情弹窗，查看具体检测项结果（如是否盖章、是否超过有效期等）

### 1.3 交互要点

- 资格审查弹窗打开 → 展示AI智能报名文件检测表格
- 检测结果下拉切换（通过/未通过）→ 异常建议列自动清空（若原值为待核实）
- 点击「移除此项」→ 该行从表格中删除
- 采购变更 → 报名截止时间更新 → 新报名截止后重新触发AI检测计算
- 资审附件组成表格中，检测状态为「通过」时显示绿色图标和文字；为「异常」时显示红色按钮，点击弹出「附件检测详情」弹窗

## ⚙️ 触发逻辑说明

### 5.1 自动计算触发时机

| 场景 | 触发时机 | 说明 |
|------|---------|------|
| 首次计算 | 报名截止时间到达后 | 系统开启智能审查功能后，报名截止自动触发AI检测 |
| 重新计算 | 采购发生变更，新的报名截止时间到达后 | 变更后需重新报名，新截止后重新触发检测 |

### 5.2 数据状态流转

```
系统检测生成
    ↓
检测结果 = 待核实（异常建议 = 系统超时提示）
    ↓
人工选择 通过/未通过 → 异常建议清空
    ↓
人工可继续调整重要程度
    ↓
可移除此项
```

### 5.3 附件智能检测规则

1. **附件检测状态**：每个附件有独立的检测状态，分为「通过」「异常」「--」（未检测）
2. **异常详情查看**：状态为「异常」时，点击「异常，查看详情」按钮，弹出详情弹窗，展示该附件的具体检测项列表
3. **检测项示例**：是否加盖公章、法人签字、有效期校验、颁发机构核验等
4. **详情弹窗内容**：包含检测项名称、检测结果（通过/未通过）、详情说明（未通过原因）

### 5.4 核心规则

1. **检测结果下拉选项**：仅允许选择「通过」或「未通过」，「待核实」为系统初始状态，不可人工选择回退
2. **异常建议清空规则**：当检测结果从「待核实」切换为「通过」或「未通过」时，`suggestion` 字段自动置空
3. **移除操作**：点击「移除此项」后，该行数据从列表中过滤删除
