## 1. 摘要

### 1.1 一句话说明

为运营人员提供对不同组织开启“数字合规官”功能的开关，并支持针对招投标各业务节点的合规校验规则进行精细化配置。

## 2. 背景与问题

### 2.1 背景

- 业务背景：在招投标系统中，采购商在招标过程中（如发布公告、编制招标文件、评标等）可能存在不合规风险。为了降低合规风险，引入“数字合规官”角色进行自动化校验。
- 用户背景：运营人员需要能够灵活地为不同的采购组织（企业/机构）开启该功能，并根据不同组织的业务特点配置不同的校验强度和规则。
- 技术/合规背景：基于预设的合规规则引擎，对业务节点产生的数据进行实时或准实时的扫描。

### 2.2 现状与痛点

| 角色/场景     | 当前流程                | 痛点                 | 影响              |
| --------- | ------------------- | ------------------ | --------------- |
| 运营人员/配置场景 | 缺乏统一配置入口，或需后台手动改库开启 | 无法针对不同组织灵活开启，配置效率低 | 响应业务需求慢，增加运维成本  |
| 采购商/业务流程  | 依赖人工审核或事后审计         | 无法在业务节点实时发现不合规信息   | 产生法律风险，可能导致招标无效 |

## 3. 用户与场景

### 3.1 目标用户（Personas）

| Persona | 关键特征           | 主要目标                        | 主要阻碍                  | 触达渠道 |
| ------- | -------------- | --------------------------- | --------------------- | ---- |
| 运营人员    | 负责系统的组织管理与功能配置 | 能够灵活地为不同采购商开启数字合规官，并进行个性化设置 | 配置项不全、无法批量控制下级、权限管控不严 | 运营后台 |

### 3.2 核心使用场景（Top Scenarios）

1. **场景A：新组织功能开启**：运营人员在后台搜索到某个采购组织 -> 点击新增配置 -> 选择该组织 -> 设置范围为“本下级” -> 勾选业务系统（如：招投标） -> 上传数字合规官头像 -> 保存生效。
2. **场景B：配置范围调整**：某组织原本只开启了“本级”，现在需要调整为“本下级”以覆盖子公司。
3. **场景C：业务系统扩展**：某组织原本只在招投标系统中使用了合规官，现在需要扩展到合同系统。

### 3.3 用户故事（User Stories）

- 作为运营人员，我想要能够选择配置的范围（本级/本下级），以便更高效地管理组织树下的合规功能。
- 作为运营人员，我想要多选业务系统（招投标/合同），以便控制数字合规官在不同系统中的服务覆盖。
- 作为运营人员，我想要为数字合规官配置个性化别名，以便在业务系统中展示更具识别度的名称。
- 作为运营人员，我想要为数字合规官配置个性化头像，以便提升产品的亲和力和品牌感。
- 作为运营人员，我需要特定的权限点才能访问配置页面，以便确保系统配置的安全性。

### 3.4 用户旅程/流程（User Flow）

1. **进入管理后台**：运营人员登录 -> 导航至“数字合规官配置”模块（需权限校验）。
2. **列表管理**：在配置列表页查看已有配置 -> 支持搜索组织 -> 点击“新增配置”。
3. **表单填写**：
   - **组织选择**：从组织树中选择一个目标组织。
   - **范围控制**：选择“本级”或“本下级”。
   - **业务系统选择**：多选框勾选（目前支持：招投标、合同）。
   - **合规人员配置**：选择目前用户中心维护的，采购商类人员。人员选择器组件应该是通用的，支持多选。
   - **定制化配置**：配置合规官别名、上传合规官头像。
4. **提交生效**：点击保存 -> 系统校验（如组织是否已配置、人员是否选择） -> 提示成功 -> 返回列表页。
