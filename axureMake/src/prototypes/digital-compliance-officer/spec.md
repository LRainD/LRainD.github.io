# 数字合规官配置

## 📋 业务与功能

### 1.1 核心目标
为运营人员提供对不同组织开启"数字合规官"功能的开关，并支持针对招投标各业务节点的合规校验规则进行精细化配置。

### 1.2 功能清单
- **配置列表页 (P-001)**：提供全局配置概览与管理入口，支持搜索、状态筛选、新增、查看、编辑、删除
- **组织及范围配置 (R2)**：实现组织维度的精准开启与层级控制，支持"本级"与"本下级"
- **业务系统多选配置 (R3)**：灵活控制合规官的服务范畴，目前包含：招投标、合同
- **合规人员关联配置 (R4)**：明确合规官背后的实际负责人，支持多选【采购商】角色人员，需展示登录账号
- **定制化配置 (R5)**：支持上传合规官头像，提升用户体验与品牌化
- **权限点管控 (R6)**：确保核心配置操作的安全性

### 1.3 交互要点
- **状态切换**：列表页开关组件点击即时生效，提示"已[开启/关闭]"
- **新增配置**：点击新增按钮跳转至表单页，选择组织、设置范围、选择系统、上传头像、保存
- **编辑配置**：点击编辑按钮跳转至表单页，编辑时不可修改组织
- **查看详情**：点击查看按钮弹出详情弹窗，所有字段只读展示
- **删除配置**：点击删除按钮二次确认后删除

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）

| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `onStatusChange` | `{ id: string, status: boolean }` | 状态开关切换时 | 状态变更通知 |
| `onConfigAdd` | `{ record: object }` | 新增配置保存成功时 | 新增配置通知 |
| `onConfigEdit` | `{ record: object }` | 编辑配置保存成功时 | 编辑配置通知 |
| `onConfigDelete` | `{ id: string }` | 删除配置成功时 | 删除配置通知 |

| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `refreshList` | `void` | - | 刷新配置列表 |
| `openAddForm` | `void` | - | 打开新增配置表单 |
| `openEditForm` | `{ id: string }` | 配置ID | 打开编辑配置表单 |
| `openViewModal` | `{ id: string }` | 配置ID | 打开查看详情弹窗 |

| 变量名称 | 类型 | 默认值 | 说明 |

| `view` | `string` | `'list'` | 当前视图：list/add/edit |
| `dataSource` | `array` | `[]` | 配置列表数据 |
| `editingRecord` | `object\|null` | `null` | 当前编辑的记录 |
