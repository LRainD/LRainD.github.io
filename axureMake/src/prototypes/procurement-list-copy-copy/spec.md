## 1. 摘要

### 1.1 一句话说明

为采购项目经办人和关联合规人员提供基于大模型意图识别的对话式合规风险查看入口，实现自然语言一键查询全节点合规风险，提升风控整改效率。

## 2. 产品概览

### 1.1 业务背景与痛点

1. 业务现状：数字合规官核心风控检测能力已完成落地，可实现招投标全流程13个核心节点的合规风险自动化校验，建立完善的组织级开关、人员管控、风控规则引擎与检测结果输出体系。
2. 核心痛点：
   - 用户侧：采购经办人查看风控结果入口深，无法通过自然语言快速获取全节点风险概览，风险检测完成后无强感知提醒，易遗漏合规问题；
   - 研发侧：团队优先落地轻量化大模型意图识别能力，基于现有大模型基座做垂直场景优化，避免复杂的多轮对话与生成式开发，同时为后续能力升级预留架构空间；
   - 业务侧：传统功能式入口无法满足用户个性化查询需求，需通过对话式交互降低合规功能的使用门槛，提升风控整改效率。
3. AI能力引入的必要性：通过大模型自然语言意图识别，实现用户自然语言指令到功能动作的精准闭环，替代传统多层级页面点击操作，同时为后续风控结果解读、合规规则咨询、整改建议生成等深度智能化能力提供基础载体。

### 1.2 核心目标

#### 一期MVP核心目标

1. 业务目标：落地轻量化对话式交互入口，基于大模型实现单意图精准识别，完成「合规风险一键查看」核心闭环，复用现有风控数据与业务接口，快速上线验证用户需求。
2. 体验目标：用户获取全节点合规风险的操作路径从≥3步缩短至1步，支持自然语言查询，风险检测完成后主动触达提醒，风险查看效率提升70%以上。
3. 架构目标：建立标准化的大模型对话系统框架，完成意图识别核心能力落地，为后续升级生成式AI问答、多轮对话、深度合规解读能力预留完整扩展位。

#### 未来及长期目标

1. 从「单意图导航助手」升级为「全场景合规智能助手」，实现风控结果解读、合规规则咨询、文本合规自检、整改建议生成等核心能力；
2. 从单意图识别升级为多意图自然语言识别，支持用户泛化表述的精准识别，覆盖多类核心业务场景，意图识别准确率≥90%；
3. 实现对话历史服务端存储、多轮上下文对话、个性化合规建议等进阶能力。

## 3. 目标用户与核心场景

### 3.1 目标用户与权限规则

| 用户角色    | 权限准入规则（必须同时满足）                                                                      | 核心需求                                              |
| ------- | ----------------------------------------------------------------------------------- | ------------------------------------------------- |
| 采购项目经办人 | 1. 所在采购组织已在运营后台开启数字合规官功能，且状态为启用；2. 为当前采购项目的直接经办人；3. 当前招采项目未处于已废标/已完成状态              | 通过自然语言快速查看当前项目全节点合规风险，接收风险检测完成提醒，快速定位预警项，查看详细检测报告 |
| 关联合规人员  | 1. 所在采购组织已在运营后台开启数字合规官功能，且状态为启用；2. 为运营后台在数字合规官配置中，为该组织关联的合规人员；3. 当前招采项目未处于已废标/已完成状态 | 快速查看所负责项目的全节点合规风险概览，接收风险提醒，跟进风险处置进度               |

### 3.2 核心用户场景

#### 场景1：自然语言查询全节点合规风险（主场景）

- 前置条件：用户已进入已开启数字合规官的采购项目业务节点页面，具备访问权限
- 用户旅程：用户点击侧边栏数字合规官头像打开聊天框 → 输入自然语言指令（如“查看当前招标的风险”“我的项目有什么合规问题”等） → 大模型识别为有效意图 → 系统拉取当前项目全节点风控数据 → 对话区输出结构化合规风险卡片 → 用户展开节点查看子项详情，点击【查看报告】获取该节点完整检测报告
- 后置条件：用户完成风险查看，可基于报告进行内容整改

#### 场景2：快捷按钮一键查看风险

- 前置条件：用户已打开聊天框，具备访问权限
- 用户旅程：用户点击聊天框底部固定的【查看合规风险】快捷按钮 → 直接触发有效动作 → 系统拉取风控数据输出结构化风险卡片
- 后置条件：用户完成风险查看

#### 场景3：检测完成后风险主动提醒

- 前置条件：用户在当前页面发起合规检测（异步预检/同步强校验），检测流程完成
- 用户旅程：系统完成全节点合规检测 → 侧边栏数字合规官自动弹出气泡提示，告知用户检测出的风险数量 → 用户点击气泡/头像，打开聊天框查看完整风险详情
- 后置条件：用户查看风险详情，完成整改

#### 场景4：非合规意图兜底处理

- 前置条件：用户打开聊天框，输入非合规风险相关内容/点击非有效意图快捷按钮
- 用户旅程：用户点击【项目状态查询】按钮/输入非有效意图的内容 → 大模型识别为无效意图 → 触发兜底规则，输出固定兜底话术，明确告知能力范围，不做任何超出边界的响应
- 后置条件：用户明确功能边界，切换至有效意图操作

### 3.3 用户旅程

1. 用户进入已开启数字合规官、且未结束的招采项目业务节点页面，看到右侧固定的数字合规官侧边栏入口；
2. 系统完成合规检测后，侧边栏自动弹出风险提示气泡；
3. 用户点击气泡/侧边栏头像，弹出AI聊天框，右侧固定展示当前项目基本信息，对话区展示动态欢迎语；
4. 用户点击【查看合规风险】快捷按钮/输入相关自然语言指令，大模型识别有效意图，拉取当前项目风控数据，输出结构化风险卡片；
5. 用户展开风险节点查看子项详情，点击【查看报告】按钮，弹窗展示该节点的完整合规检测报告；
6. 用户关闭聊天框，未刷新再次打开时，记录保留；刷新页面/项目结束后，历史对话全部清空，重置对话，仅展示欢迎语。

## 4. 核心AI能力定义（AI-PRD核心模块）

### 4.1 一期MVP AI能力边界

#### 4.1.1 核心能力（仅支持1项主能力）

**大模型驱动的单意图精准识别能力**：仅支持「查看当前项目全节点合规风险」这唯一有效意图，通过固定快捷按钮+大模型自然语言意图识别两种方式触发。大模型仅做二分类意图判定（有效意图/无效意图），触发有效意图后自动拉取当前项目风控数据，输出标准化结构化风险卡片，一期不做任何生成式内容创作、开放式问答、多轮对话。

#### 4.1.2 能力触发规则

仅以下两种方式可触发有效能力，其余所有操作全部触发兜底策略：

1. 快捷按钮触发：用户点击聊天框底部固定的【查看合规风险】快捷按钮，直接触发有效动作，无需经过大模型意图识别；
2. 大模型意图识别触发：用户输入任意自然语言内容，系统自动调用大模型进行意图二分类识别，仅当大模型判定结果为「查看当前项目合规风险」有效意图时，触发对应动作；其余所有识别结果，触发兜底策略。
   一期大模型识别范围限定：仅识别用户是否有查看当前项目合规风险的诉求，包括但不限于“查看风险、合规问题、哪里不合规、检测结果、预警情况”等泛化表述，不支持其他任何意图识别。

#### 4.1.3 绝对禁止能力（红线规则，研发严格执行）

1. 禁止响应任何与「查看当前项目合规风险」无关的用户意图，包括但不限于项目状态查询、流程操作咨询、通用闲聊等；
2. 禁止任何生成式文本创作，除固定话术与标准化风控卡片外，禁止大模型自由生成任何文本内容；
3. 禁止调用任何当前项目以外的业务数据、风控数据，禁止跨组织、跨项目、跨节点的数据查询；
4. 禁止任何违反招投标法律法规、企业合规管理要求的内容输出；
5. 禁止一期开放多意图识别能力，仅保留单意图二分类判定能力。

### 4.2 二期及未来AI能力规划（预留扩展位）

1. 意图识别能力升级：从单意图二分类升级为基于NLU的多意图自然语言识别，支持用户泛化表述的精准识别，覆盖「规则解读、风险解析、整改建议、文本自检」等多类核心意图；
2. 生成式问答能力：基于招投标合规知识库+当前项目风控数据，实现合规问题的精准生成式解答，包括风险原因解读、整改建议生成、合规规则释义等；
3. 多轮对话能力：支持上下文关联的多轮对话，可针对单个风险点进行连续追问、深度答疑，实现槽位填充与意图流转管理；
4. 文本自检能力：支持用户主动发起表单文本合规自检，提前识别潜在合规风险，给出优化建议。

## 5. 对话系统核心规则（AI-PRD核心模块）

### 5.1 对话生命周期管理

1. 会话初始化：用户每次打开聊天框，视为一次全新会话，自动清空上一次对话历史，仅展示动态欢迎语；
2. 会话存续期：仅在当前页面不刷新、不关闭、项目未结束的情况下，保留本次会话的对话历史（前端内存级存储）；
3. 会话终止：页面刷新、关闭标签页、切换项目、项目状态变更为已结束、关闭聊天框超过5分钟重新打开，会话自动终止，历史对话全部清空。

### 5.2 输入输出规范

#### 5.2.1 用户输入规则

1. 输入格式：支持多行文本输入，最大输入长度500字，超出后无法继续输入；
2. 空输入处理：输入内容为空时，发送按钮置灰不可点击，按下回车键无响应。

#### 5.2.2 AI输出规则

| 输出类型      | 输出规范                                                                                                                                                                                                                | 触发场景                                         |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 动态欢迎语     | 1. 无风险项目：您好，我是您的数字合规官。今天是{{当前日期}}，请问有什么可以帮您？您可以点击【查看合规风险】或输入相关问题，获取项目合规情况。2. 存在风险项目：您好，我是您的数字合规官。检测到您当前项目存在合规风险，您可以点击【查看合规风险】或输入相关问题，查看风险详情。3. 检测中项目：您好，我是您的数字合规官。当前项目部分节点正在进行合规检测，您可以点击【查看合规风险】或输入相关问题，查看已完成的检测结果。 | 每次打开聊天框，会话初始化时输出                             |
| 侧边栏风险提示气泡 | 1. 存在风险通用文案：检测到您有{{风险总数}}项合规风险2.  无风险文案：本次检测未发现合规风险                                                                                                                                                                 | 异步预检/同步强校验完成后触发                              |
| 结构化风险卡片   | 严格按照通用渲染规则输出，适配所有采购类型的业务节点：1. 一级节点使用风控规则配置时的「业务环节」字段；2. 二级节点使用风控规则配置时的「场景名称」字段；3. 采购类型项目需进行招标到采购的名称替换（如「定标」→「成交」）；4. 包含节点状态、风险等级、子项详情、查看报告按钮等元素                                                                     | 有效意图触发后，成功拉取到风控数据时输出                         |
| 固定兜底话术    | 我尚未掌握此信息。作为数字合规官，我主要协助识别与应对数据、隐私及监管相关的合规风险，您可以询问当前项目的合规风险相关问题。                                                                                                                                                      | 大模型识别为无效意图、用户点击非有效意图按钮时，100%输出该固定话术，无任何个性化修改 |
| 异常提示话术    | 1. 网络异常：「网络异常，无法获取合规数据，请稍后重试」2. 无数据：「当前项目暂无合规检测数据」3. 无权限：「您暂无当前项目的合规数据查看权限」4. 大模型调用异常：「服务暂时无法响应」                                                                                                                    | 对应异常场景触发时输出                                  |

### 5.3 兜底策略与异常处理

#### 5.3.1 全量兜底策略（无任何例外）

1. 所有不在有效触发规则内的用户操作，包括但不限于大模型识别为无效意图的输入内容、点击非有效意图按钮、询问超出能力边界的问题，100%触发固定兜底话术，不做任何其他处理；
2. 兜底话术仅支持固定文本输出，禁止任何泛化、生成式修改；

## 6. 交互与功能需求

### 6.1 侧边栏AI入口

#### 6.1.1 显隐规则

**同时满足以下4个条件时展示，否则完全隐藏**：

1. 当前页面为**招标列表、询价列表、采购列表**对应的业务页面；
2. 登录用户所在采购组织已在运营后台开启数字合规官功能，且功能状态为「启用」；
3. 登录用户为当前采购项目的经办人，或为该组织在运营后台配置的关联合规人员；
4. 当前招采项目的主状态为进行中，招标/采购结束后，侧边栏入口自动隐藏。

**数据来源**：招采状态数据来自招投标业务系统，组织配置与人员数据来自风控预警平台

#### 6.1.2 展示与交互规则

1. 展示位置：业务页面右侧固定悬浮，不遮挡表单核心操作区，层级高于页面普通元素，低于弹窗，与页面现有侧边工具栏（联系客服、权限助手等）合并展示；
2. 头像展示规则：**读取《数字合规官》配置文档中，当前招标/采购所属组织配置的数字合规官定制化头像**；
3. 别名展示规则：**读取《数字合规官》配置文档中，当前招标/采购所属组织配置的合规官别名，展示在侧边栏头像下方或悬浮提示中**；
4. 基础交互规则：点击头像弹出/关闭聊天框；

#### 6.1.3 风险提示气泡规则（对应截图需求）

1. **触发条件**：当前页面发起的异步预检/同步强校验流程全部完成后，无论是否存在风险，均触发气泡提示；仅在当前页面停留时触发，页面刷新/关闭后不重复触发；
2. **展示位置**：气泡位于侧边栏数字合规官头像左侧，带指向箭头，不遮挡页面核心操作区；
3. **文案规范**：
   - 存在风险时：默认展示「检测到您有{{风险总数}}项合规风险」；
   - 无风险时：展示「本次检测未发现合规风险」；
4. **展示与消失规则**：
   - 气泡弹出后，**若存在风险则持续展示，不会自动消失**；
   - 用户点击气泡/头像任意位置，气泡立即消失，同时打开聊天框；
5. **重复触发规则**：同一次检测结果仅触发一次气泡提示，用户重新发起检测后，按最新检测结果更新气泡文案并重新触发。

### 6.2 聊天框主体与风险卡片渲染

#### 6.2.1 聊天框基础结构

采用「左对话区+右项目信息栏」固定布局，整体尺寸固定，支持拖动，不支持缩放：

1. 顶部栏：左侧展示数字合规官logo+别名+在线状态，右侧关闭按钮，点击关闭聊天框；别名读取自《数字合规官》配置文档中当前招标/采购所属组织配置的合规官别名；
2. 左侧对话区：从上到下依次为对话历史展示区、底部输入框、固定快捷操作按钮区；
3. 右侧项目信息栏：固定展示采购名称、采购编号、组织机构、采购方式、采购项目、采购品类、经办人，与当前项目强绑定，项目切换时自动更新。

#### 6.2.2 快捷操作按钮规则

1. 固定展示在输入框下方，全程可见，不随对话滚动；
2. 一期固定展示2个按钮：【查看合规风险】、【项目状态查询】；
3. 【查看合规风险】：点击后直接触发有效意图，输出风险卡片，无需经过大模型识别；
4. 【项目状态查询】：点击后触发兜底策略，输出固定兜底话术。

#### 6.2.3 合规风险卡片通用渲染规则（适配全采购类型）

1. 数据来源：风控预警中心；
2. 节点名称规范：
   - **一级节点名称**：使用风控规则配置时的「业务环节」字段，作为一级卡片标题展示；
   - **二级节点名称**：使用风控规则配置时的「场景名称」字段，作为二级子项标题展示；
3. 采购类型名称替换逻辑：
   - 当当前项目为**采购类型**时，需对业务环节和场景名称进行招标到采购的名称替换，如「定标」→「成交」；
   - 名称替换需基于项目的采购方式属性动态判断，确保与业务系统术语一致；
4. 排序规则：与业务系统采购流程进度顺序100%一致；
5. 展开规则：默认所有节点折叠，存在预警/拦截风险的节点默认自动展开，用户可手动切换展开/折叠状态；
6. 【查看报告】按钮规则：节点为「报名与资审汇总、查看报价情况、报价汇总、成交报告」点击后跳转至风控报告页，展示对应节点的完整合规检测报告。

## 7. 数据与模型要求

### 7.1 数据来源规范

1. 权限与配置数据：来自运营后台《数字合规官》配置模块的组织功能开关、关联合规人员、定制化头像数据；
2. 项目基础数据与状态数据：来自招投标业务系统的项目详情接口、项目主状态接口；
3. 风控检测数据：来自风控预警中心的全节点风控检测结果接口、合规报告详情接口；
4. 所有数据仅在当前项目会话内调用，禁止跨项目、跨组织数据调用。

### 7.2 模型与调用要求（一期MVP）

1. 模型选型：优先使用公司现有可调用的大模型基座，基于垂直场景做Prompt优化，仅实现二分类意图识别能力，无需额外模型训练，最大化降低研发成本；
2. Prompt工程规范：必须严格限定大模型的输出范围，Prompt需包含以下核心约束：
   - 任务：仅对用户输入的文本进行二分类意图判定，判断用户是否想要查看当前招投标项目的合规风险、检测结果、预警问题；
   - 输出格式：仅允许输出「有效」或「无效」，不允许输出其他任何内容；
   - 约束：禁止生成任何其他文本内容，禁止解读用户输入，禁止回答用户问题，仅做二分类输出；
   
### 7.3 二期模型预留要求

1. 预留RAG知识库调用接口，支持后续接入招投标合规知识库；
2. 预留大模型生成式对话接口，支持复杂Prompt工程配置、模型参数调整、输出内容审核；
3. 预留多意图识别模型接口，支持多意图分类、实体抽取、槽位填充能力扩展。
